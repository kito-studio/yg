## 1. プロジェクトの狙い

- 事務的で地味なタスクを「撃破対象」に変換し、達成行動を継続しやすくする。
- きとスタ本体とは分離して先行リリースし、UI/演出の実験場として活用する。
- マップ配布で他者の計画を取り込み、最短で実行モードに入れるようにする。

## 2. コンセプト

褐色の大地に潜む魔物としてタスクを配置し、重厚な演出で撃破して進捗を可視化する。

- しょぼいタスクほど派手に倒す。
- 実行行動を最優先し、設定は後から整えられる。
- 自作素材（画像・動画・音源）で世界観を拡張できる。

想定用途例:

- 3年6組 2026年夏休みの宿題攻略
- 行政書士合格ロードマップ
- Instagramフォロワー1万人達成プラン

## 3. システム方針（ローカルファースト）

- プラットフォーム: PWA対応Webアプリ（PC/Android Chrome想定）
- 技術スタック: TypeScript / Vite / Tailwind CSS
- 描画: Three.js（背景・演出） + Canvas/SVG（マップUI）
- 保存: IndexedDB（ログイン不要、端末ローカル保存）
- 共有/同期: JSONインポート・エクスポート
- 暗号化: Web Crypto API（AES、同期時の機密保護用）

## 4. データモデル（最小単位）

- ステージ
- マップ
- 親タスク
- タスク

各ノードに「撃破前/撃破後」の見た目を持たせる。

## 5. MVP機能（先行リリース対象）

1. ステージ/マップ/タスクの作成・編集
2. 背景上への自由配置（ドラッグ&ドロップ）
3. 依存関係の線表示（任意）
4. タスク進捗入力（撃破率）
5. 完了時の撃破演出（SE・パーティクル）
6. マップJSONの入出力

## 6. 拡張機能（MVP後）

- 第三者承認トークン（完遂確認コード）
- 救出・カードコレクション
- シークレット報酬（達成まで非公開）
- マップ共有導線（SNS配布テンプレ）
- 収益導線（任意視聴型リワード広告）

## 7. 開発マイルストーン（1週間）

1. DB基盤: YGデータ保存スキーマとCRUD
2. 基本マップ: 背景 + ノード配置 + 接続線
3. 演出基盤: 撃破エフェクト + BGM/SE再生制御
4. 共有基盤: JSON入出力 + AES暗号化オプション
5. 報酬要素: カード解放 + シークレット表示

## 8. きとスタ開発での活用方法

1. きとスタ開発タスクを「ステージ > マップ > 親タスク > タスク」に分解して登録する。
2. 毎日最初に「今日撃破する3タスク」を実行モードで固定する。
3. 完了時は必ず撃破演出を通して、達成ログを残す。
4. 週末にマップをJSON出力して進捗レビューする。
